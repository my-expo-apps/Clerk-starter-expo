## איפיון טכני (High-level)

### משתני סביבה (Client-side)

- **Clerk**:
  - `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY`
- **Supabase**:
  - `EXPO_PUBLIC_SUPABASE_URL`
  - `EXPO_PUBLIC_SUPABASE_ANON_KEY`

> הערה: אלו מפתחות “ציבוריים” (client keys). לא שמים בריפו שום secret.

### קבצים מרכזיים

- `config/clerk.ts`: דרישת publishable key בזמן ריצה
- `config/supabase.ts`: דרישת URL + anon key בזמן ריצה
- `lib/supabase.ts`: יצירת Supabase client + polyfill + AsyncStorage

### מסכים (Routes)

- `/(home)`:
  - כניסה/יציאה + קישורים למסכי Supabase
- `/(auth)`:
  - `/auth` מסך אחד עם טאבים
- `/(supabase)`:
  - `/setup` הנחיות + “Test connection”
  - `/schema-designer` UI ליצירת טיוטת טבלה וייצוא SQL

### DB Schema (Supabase)

- `supabase/migrations/001_init.sql`:
  - `profiles`, `projects`, `schema_drafts`
  - triggers ל־`updated_at`
  - RLS policies בסיסיות (מניחות Supabase Auth כברירת מחדל)

### מה חסר עדיין (בכוונה)

- אינטגרציה מלאה Clerk↔Supabase Auth (JWT custom / third-party auth) — זה שלב מתקדם שיוגדר בהמשך לפי צורך.
