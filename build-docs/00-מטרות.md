## מטרות

- **אימות משתמשים**: כבר קיים Clerk (Sign-in / Sign-up) עם מסך אחוד `/auth`.
- **הכנה ל‑Supabase DB**: הוספת תשתית לקונפיגורציה + לקוח Supabase + סכמות בסיס + UI פנימי לבניית סכמות לאפליקציות עתידיות.
- **אפס סודות בריפו**: שימוש רק ב־`EXPO_PUBLIC_*` בצד לקוח. אין Commit ל־`.env`.

## עקרונות

- **Separation of Concerns**: קונפיג ב־`config/`, לקוחות/SDK ב־`lib/`, UI במסכים ב־`app/`.
- **Future-proofing**: שמירת טיוטות סכמות בפורמט JSON (בעתיד אפשר לייצר migrations אוטומטית).
- **Simple first**: סכמות בסיסיות + UI קל לייצוא SQL, בלי תלות ב‑backend.
