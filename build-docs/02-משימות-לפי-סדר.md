## משימות לפי סדר הבנייה (Roadmap)

### שלב 0 — בסיס (כבר קיים)
- Clerk auth עובד
- מסך `/auth` עם טאבים
- Dark theme כחול-כחלחל

### שלב 1 — Supabase Config
- להגדיר `.env` מקומי עם:
  - `EXPO_PUBLIC_SUPABASE_URL`
  - `EXPO_PUBLIC_SUPABASE_ANON_KEY`
- לוודא שהאפליקציה עולה בלי שגיאות קונפיג

### שלב 2 — יצירת פרויקט Supabase + סכמות
- ליצור פרויקט ב‑Supabase
- להריץ `supabase/migrations/001_init.sql` ב‑SQL Editor
- לוודא RLS פעיל

### שלב 3 — UI “Schema Designer”
- ליצור טבלה ו־fields דרך המסך
- לייצא SQL ולבדוק שהוא רץ ב‑Supabase
- לשפר UI:
  - בחירת סוגי שדות מ‑dropdown במקום input חופשי
  - validations לשמות מזהים

### שלב 4 — Persist Drafts (עתידי)
- לשמור טיוטות ב‑Supabase (`schema_drafts.definition`)
- CRUD למסכים

### שלב 5 — Clerk ↔ Supabase (עתידי מתקדם)
- החלטה על אסטרטגיית auth:
  - Edge function שמחליפה Clerk JWT ל‑Supabase JWT
  - או שימוש ב‑Supabase Custom JWT
- התאמת RLS לפי מזהה משתמש מ‑Clerk
